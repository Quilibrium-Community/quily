**Quilibrium Node v2.1.0.23 is live.**

This release marks the completion of the full rewrite from Go to Rust. The new node delivers significant performance and memory improvements alongside first-class libp2p, RocksDB, and async-runtime integration.

Release files: https://releases.quilibrium.com/release

**Updated documentation:**

- Quick Start: https://docs.quilibrium.com/docs/run-node/quick-start
- Linux Configuration: https://docs.quilibrium.com/docs/run-node/linux_configuration
- Linux Manual Install: https://docs.quilibrium.com/docs/run-node/node-provisioning/manual-install/linux
- macOS Manual Install: https://docs.quilibrium.com/docs/run-node/node-provisioning/manual-install/macos

**IMPORTANT — file descriptor limits**

The Rust node opens significantly more file descriptors than the Go version did (one per RocksDB SST, one per libp2p peer, one per worker IPC channel). The default Linux `ulimit -n` of `1024` and the default macOS `kern.maxfiles` are well below what the node needs and WILL cause RocksDB open failures and dropped peer connections under load.

Please apply the tuning steps for your platform before running (if you have not already):
- Linux file descriptors: https://docs.quilibrium.com/docs/run-node/linux_configuration#file-descriptor-limits-ulimit--n
- Linux kernel network tuning: https://docs.quilibrium.com/docs/run-node/linux_configuration#kernel-network-tuning
- macOS file descriptors: https://docs.quilibrium.com/docs/run-node/node-provisioning/manual-install/macos#3-raise-the-file-descriptor-limit
