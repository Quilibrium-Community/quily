# Community Recap — #general

**June 21, 2026** | 12:31 AM UTC – 9:31 AM UTC

## Archive Node Issues & Infrastructure
Users discussed operational problems with archive nodes. Danske reported that their archive server went down, but later confirmed the provider resolved the issue and the node would be back up soon. glatki operater noted another archive node at the same provider had zero incidents, suggesting variability.

Finn asked whether all archive nodes run on rented hardware or if anyone runs them from home. blacks1ne responded that the 10 Gbps bandwidth requirement makes home setups infeasible, so all archive nodes—whether rented or owned—are deployed in data centers.

## Decentralized AI Project Comparison
Otter shared a GitHub link for a decentralized AI project that uses sharding. blacks1ne clarified that project (c0mpute.ai) is a product-first network aiming to aggregate idle gaming GPUs into a virtual supercomputer for private, uncensored AI inference. This is distinct from Quilibrium's broader vision, which is CPU-centric. andree asked if the projects are similar, and blacks1ne explained the differences.

## Role of GPUs in Quilibrium's Future
yogurt asked whether GPUs will play a defined role in Quilibrium's future. blacks1ne summarized Cassie's previous stance: whether GPUs offer an edge for specific Quilibrium use cases remains an open question pending experiments. So far, no GPU-favoring use cases have been identified. blacks1ne noted that some node operators used GPUs in an earlier phase, but that was exploiting a past mainnet weakness, not a sustainable future scenario.
