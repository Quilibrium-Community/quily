# Community Recap — #general

**July 6, 2026** | 1:52 AM UTC – 9:15 AM UTC

## Network Status Update

The Quilibrium network is progressing through the v2.1.0.25 upgrade. Cassie [LEAD DEV] reported that nodes are becoming active and she is monitoring the pending set clearing. She is also working on qclient improvements to make network state easier to understand and investigating reported seniority = 0 cases.

## Shard Health and Risk Metrics

A significant discussion emerged around shard assignment and halt risk metrics. Cassie clarified that in version .25, the total number of shards required is the **square root** of what it was in .24, making the halt risk and warning level shards accurate.

Quilscan explained the distinction between "unassigned" shards (invalid/empty with no size and no workers) and "halt risk" shards (those with size > 0 and fewer than 3 active workers). If active = 0 and size > 0, it counts as halt risk, not unassigned.

Mr.Quil noted that halt risk dropped dramatically from around 2,000 to just 42 very quickly, contrasting with previous weeks where it would only decrease by 10 numbers over three months. This was seen as a positive sign that the upgrade is working.

## Data Inconsistency

alcohol pointed out inconsistencies between the dashboard (showing 30+ unassigned shards) and Franc's website (showing all shards as assigned). Only 49 shards currently have storage.

## QClient Troubleshooting

Alternate received help from Quilscan resolving a qclient issue on Ubuntu 24.04/x86_64. The fix involved updating to the latest qclient version and ensuring `listenGrpcMultiaddr` is properly configured in the node config.yml.

## Current Status

Pompey the Great asked about the current release status. Aza confirmed that v2.1.0.25 is still being tested, with first round shard progress expected after approximately 2,160 frames for workers to fully confirm as active.
