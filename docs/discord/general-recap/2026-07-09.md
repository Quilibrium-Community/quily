# Community Recap — #general

**July 9, 2026** | 12:50 AM UTC – 8:51 AM UTC

## Network Performance and .25 Release

Nikos148 reported that their peer ID shows no Stream Multiaddrs or Key Registry on the dashboard, though the node appears to be running fine. Finn❄❄❄ noted that all shards appear healthy, but bansabai pointed out 25-minute frame gaps in performance.

## From Cassie

Cassie addressed the 25-minute gaps, explaining they are caused by more peers now running version .25, which increases RPC pressure on hypersync. Unlike the 4096 shard era, there are no file store copies to pull from, leading to timeouts. She outlined two fixes in progress:

1. Separating resource allocation on the consensus path from hypersync so they cannot interfere with each other
2. Adding a more persistent RPC for the consensus path

Cassie clarified that the slowdown is specific to archive nodes, which must run consensus while serving the full 144GB initial network state. She emphasized this is a sign the system is working — more nodes on .25 means more sync pressure on archives, revealing bottlenecks that can be improved through better RPC separation. This issue is not a blocker for mainnet readiness, as it only affects archives.
